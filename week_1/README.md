# Week 1 – Data Structures and Algorithms

## 📚 Overview

This week introduces the fundamental concepts of **Data Structures and Algorithms (DSA)**, which are essential for writing efficient and scalable code. Understanding how data is organized and how algorithms operate on that data will significantly improve problem-solving skills and performance optimization.

---

## 🧱 Linear Data Structures

Linear data structures store elements in a sequential manner, where each element is connected to its previous and next element.

### Arrays

* A collection of elements stored in contiguous memory locations.
* Provides fast access using indices (O(1)).
* Ideal for scenarios where quick lookup is required.

### Lists (Linked Lists)

* A sequence of nodes where each node points to the next.
* Efficient for insertions and deletions.
* Slower access time compared to arrays (O(n)).

### Queues

* Follows FIFO (First In, First Out) principle.
* Elements are added at the rear and removed from the front.
* Commonly used in task scheduling and buffering.

### Stacks

* Follows LIFO (Last In, First Out) principle.
* Elements are added and removed from the top.
* Used in recursion, undo operations, and expression evaluation.

---

## 🌳 Non-Linear Data Structures

Non-linear data structures do not store data sequentially. Instead, they represent hierarchical or interconnected relationships.

### Map

* Stores key-value pairs.
* Each key is unique and maps to a specific value.
* Useful for fast data retrieval.

### Hash Tables

* Uses a hash function to compute an index for storing values.
* Offers average O(1) time complexity for search, insert, and delete operations.

### Set

* Stores unique elements only.
* Useful for eliminating duplicates and membership checks.

### Trees

* Hierarchical structure with a root node and child nodes.
* Common types include Binary Trees and Binary Search Trees.
* Used in databases, file systems, and hierarchical data representation.

### Graphs

* Consists of nodes (vertices) and edges (connections).
* Can be directed or undirected.
* Widely used in networks, pathfinding, and recommendation systems.

---

## ⚙️ Algorithms

Algorithms define step-by-step procedures to solve problems efficiently.

### Binary Search

* Works on sorted data.
* Repeatedly divides the search interval in half.
* Time complexity: O(log n).

### Depth-First Search (DFS)

* Explores as far as possible along each branch before backtracking.
* Commonly implemented using recursion or a stack.
* Used in pathfinding and cycle detection.

### Breadth-First Search (BFS)

* Explores all neighbors at the current depth before moving deeper.
* Implemented using a queue.
* Useful for finding the shortest path in unweighted graphs.
